Week 10 Guide

Chapter 7 — Data Cleaning and Preparation

Modified

March 15, 2026

Week 7 introduced pd.read_csv() and the broader family of pd.read_*() functions that replace the manual parsing pipeline with a single function call: open the file, detect the header, infer column types, and return a structured DataFrame. But a loaded DataFrame and an analysis-ready DataFrame are not the same thing. pd.read_csv() faithfully represents what is in the file, including the problems. Real data almost always has them.

Chapter 7 covers the work that happens between loading and analyzing. Data from real sources arrives with missing values where records were never filled in, columns that loaded as text because one placeholder string blocked type inference, duplicate rows from logging errors or data pipeline issues, and string columns where the same value appears with different capitalization or extra whitespace. These are not edge cases. They are normal characteristics of data collected from systems that were not designed with analysis in mind. Cleaning that data before you calculate anything is what makes the results trustworthy.

The Week 10 demo works through a fleet registry dataset that has all of these problems. The demo is organized into three parts: handling missing data, data transformation, and string cleaning. Chapter 7 in the textbook goes further than the demo demonstrates, covering forward and back fill strategies for fillna(), value substitution with replace(), regular expressions for pattern-based string manipulation, and categorical data. The full chapter is worth reading alongside the demo.

Chapter 7 concepts used this week

The gap between loaded and ready

When pd.read_csv() loads a file, it reads what is there without evaluating whether the data is correct. A column where two rows say UNKNOWN instead of a number loads as a text column, because that is what the file contains. A column where one row is blank loads with a NaN in that position. Duplicate rows appear twice because they appear twice in the file. The loading function is doing its job correctly; the problems are in the source data.

This means cleaning is always a distinct step in the workflow, separate from loading. Printing the raw file contents before calling pd.read_csv() is a useful habit. Seeing problems in the raw file confirms they came from the source, not from anything pandas did during loading. After loading, .info() and isnull().sum() show what pandas made of the file and where the gaps are.

How pandas represents missing data

pandas uses NaN (Not a Number) as its sentinel value for missing data. NaN is a floating-point concept, which has an important consequence: any integer column that contains even one blank field cannot stay as an integer type, because the integer type has no representation for “missing.” pandas widens the entire column to float64 to accommodate NaN. If you load a file with a crew size column that has one blank row, that column will show float64 in .info() and all values will display as floats, even the ones that were originally whole numbers. This is a design constraint, not an error.

isnull() returns a boolean result for every cell in the DataFrame, True where a value is missing and False where it is not. Calling .sum() on that result counts the missing values per column. This is useful, but it only detects genuine missing data. A column that contains the string UNKNOWN where a number should be will show zero missing values from isnull(), because UNKNOWN is a valid string. pandas has no way to know it represents the absence of a value. Identifying and handling sentinel strings requires a different approach, covered in the type conversion section below.

Removing versus filling missing data

Once you know where missing values are, you have two choices: remove the rows that contain them, or fill the gaps with a reasonable value.

dropna() removes rows containing missing values. Its default behavior is aggressive: any row with at least one missing value in any column is dropped, even if the missing field is not important to your analysis. The subset parameter limits which columns can trigger a row being dropped. dropna(subset=['crew_size']) only removes rows where crew_size is missing, preserving rows that are missing only less critical fields like launch_year.

fillna() replaces missing values with a value you provide. Filling with a constant is simple but often misleading. A launch year of 0 would be a worse representation of the data than a blank. For numeric measurements, filling with the column mean or median is usually more appropriate. The mean is sensitive to outliers: if one value is far outside the typical range, it pulls the mean away from what a representative fill should be. The median is resistant to outliers, because it is the middle value when sorted and does not shift based on extreme values at the ends. When a column has outliers, the median is the safer fill choice.

Type conversion with pd.to_numeric()

When a numeric column contains even one non-numeric string, such as a placeholder like UNKNOWN, a corrupted record, or a formatting artifact, pandas treats the entire column as text during loading. The column appears as object or str in .info(), and you cannot perform calculations on it until it is converted.

pd.to_numeric() converts a Series to a numeric type. The errors parameter controls what happens when a value cannot be converted. The default behavior (errors='raise') throws an exception when it encounters a non-numeric value, which would stop execution. errors='coerce' handles the problem differently: any value that cannot be converted becomes NaN instead of causing an error. After the conversion, the column is a numeric type and the unconvertible strings are now missing values that can be filled using fillna().

This connects to Week 7’s na_values parameter in pd.read_csv(). Both tools solve the same underlying problem: values in the file that cannot be used as-is, but at different stages. na_values addresses the problem during loading by telling pandas which strings to treat as missing. pd.to_numeric(errors='coerce') addresses it after loading, when the column has already arrived as text.

Removing duplicates, recoding values, and renaming columns

These three operations are housekeeping: they make the DataFrame accurate and easier to work with before analysis begins.

drop_duplicates() removes rows that are exact copies of other rows, keeping the first occurrence by default. The subset parameter limits which columns define a duplicate. For example, drop_duplicates(subset=['ship_name']) removes any row where the ship name has already appeared, even if other columns differ. The default behavior compares all columns, which is the right choice when a true duplicate means every field is identical.

map() replaces existing values in a column using a dictionary lookup. It is useful when a column stores compact codes that are not readable in analysis output. Replacing ACT, RET, and MNT with Active, Retired, and Maintenance makes the data self-documenting. Any value in the column that is not a key in the dictionary becomes NaN, so the dictionary needs to cover every value present. This is different from fillna(), which fills blank cells. map() replaces non-null values that are already there.

rename() changes column names using a dictionary where each key is the current name and each value is the new name. You only need to include the columns you want to rename; the rest are left unchanged.

Detecting and replacing outliers

An outlier is a value that is technically present but so far outside the typical range that it distorts any calculation that uses it. The mean is the most common casualty: one extreme value can pull it far enough from the typical range that any calculation depending on it produces misleading results. This matters especially when the plan is to fill missing values with the column mean, because an uncorrected outlier makes the fill just as wrong as the original problem.

describe() is the diagnostic tool. It returns count, mean, standard deviation, min, max, and quartiles for a numeric column. When the max is 9999 and the 75th percentile is 120, the outlier is immediately visible. Boolean indexing then locates the specific row, and .loc assignment replaces the value in place: df.loc[condition, 'column'] = value selects the matching rows and replaces only the specified column. Replacing the outlier with NaN first, then filling with the median, is cleaner than replacing it with the fill value directly. It keeps the two decisions separate: what is wrong, and what to replace it with.

Binning continuous values with pd.cut()

Some analyses are easier with categories than with raw numbers. Rather than working with individual crew sizes, it may be more useful to classify ships as Small, Medium, or Large. pd.cut() creates those categories from a continuous numeric column.

pd.cut() takes a column and a list of bin edges, and assigns each value to the interval it falls in. The labels parameter names the intervals. Because labels name the spaces between edges rather than the edges themselves, there is always one fewer label than there are edges: three edges create two intervals, four edges create three intervals. Values that fall outside all defined bin edges receive NaN rather than being placed in the nearest bin. This is a signal that your bin edges do not cover the full range of your data.

This is why outliers must be addressed before binning. A value of 9999 with bin edges of 0, 30, 100, and 300 would receive NaN from pd.cut() because 9999 exceeds the largest edge. Removing or replacing the outlier first ensures that every valid value lands in a meaningful category.

String cleaning with the str accessor

String columns in real data are rarely consistent. The same ship class might appear as cargo, Cargo, CARGO, or cargo depending on who entered the record and whether they followed any formatting convention. To pandas, those are four different values. Any grouping, filtering, or counting operation on that column will treat them as separate categories and produce wrong results.

pandas provides a str accessor that applies string methods to every value in a Series column at once, without a loop. Methods you have used on individual strings throughout the course, including strip(), lower(), upper(), and title(), are all available through the str accessor, applied column-wide.

The order of operations matters here. str.strip() removes leading and trailing whitespace from each value. It should run first, because a value like Cargo with surrounding spaces is a different string from Cargo, and case standardization alone would not make them match. After stripping, str.lower() converts every value to lowercase, making all variations of the same class name identical. str.contains() then returns a boolean Series showing which rows include a given substring, which works correctly with boolean indexing to filter the DataFrame. If you filter before standardizing case, a search for 'cargo' will miss 'Cargo' and 'CARGO'. Cleaning in the wrong order does not produce an error. It produces results that are silently incomplete.

Reading expectations for Week 10

As you read Chapter 7 and work through the demo, check whether you can explain the following in your own words:

  1. What is the difference between a DataFrame that has been loaded and one that is ready to analyze?
  2. Why does a numeric column become float64 when it contains one blank field?
  3. Why does isnull() report zero missing values for a column that contains UNKNOWN?
  4. What is the difference between dropna() with no arguments and dropna(subset=[...])?
  5. When is the median a better fill value than the mean, and why?
  6. What does pd.to_numeric(errors='coerce') do when it encounters a value that cannot be converted to a number?
  7. What is the difference between map() and fillna()?
  8. What does pd.cut() assign to a value that falls outside all defined bin edges?
  9. Why must case standardization happen before filtering a string column with str.contains()?

Knowledge goals for Week 10

By the end of Week 10, you should be able to:

  • identify missing values with isnull() and explain why sentinel strings do not register as missing
  • choose between dropna() and fillna() based on what the missing data represents, and select an appropriate fill value
  • convert a text column to a numeric type using pd.to_numeric() and explain what errors='coerce' does
  • remove duplicate rows and recode values using drop_duplicates(), map(), and rename()
  • detect and replace outliers using describe(), boolean indexing, and .loc assignment
  • clean string columns using the str accessor and explain why the order of strip, case standardization, and filtering matters
