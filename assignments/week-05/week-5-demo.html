Week 4: Integration - Complete workflow: file → parse → clean → NumPy arrays → analysis - Combining Week 2 parsing with Week 3 NumPy operations

This Week: pandas - Built on NumPy arrays but adds powerful features for working with labeled, tabular data - If NumPy is “efficient arrays,” pandas is “efficient spreadsheets with labels”

What is pandas?

pandas provides two core data structures:

Series: A one-dimensional labeled array - Think of it as a NumPy array with labels (an index) attached - Like a single column in a spreadsheet - Example: A list of temperatures with dates as labels

DataFrame: A two-dimensional labeled data structure with columns - Think of it as a table or spreadsheet - Each column is a Series - Both rows and columns have labels - Example: A table of student grades with student names as row labels and assignments as column labels

Why pandas Exists

NumPy is powerful for numerical operations, but it has limitations when working with real-world data:

NumPy limitations: - No built-in way to label rows and columns - Selecting data requires remembering numeric positions - All data must be the same type (all floats, all ints) - No automatic alignment when combining datasets

pandas solutions: - Row and column labels make data self-documenting - Select data by meaningful names instead of positions - Different columns can have different types (one column of numbers, another of text) - Automatic alignment by labels when combining data

pandas is built on top of NumPy arrays, so you still get NumPy’s speed. But you also get labels, mixed types, and powerful data manipulation tools.

Part 1: Creating and Using Series

A Series is a one-dimensional array with labels. It’s like a NumPy array, but each value has an associated label (index).

Creating a Series from a list

import numpy as np
import pandas as pd

# Create a Series from a list
temperatures = pd.Series([72, 75, 68, 70, 73])
print(temperatures)
0    72
1    75
2    68
3    70
4    73
dtype: int64

Understanding the output:

The left column (0, 1, 2, 3, 4) is the index - the labels for each value. By default, pandas uses sequential integers starting at 0, just like list positions.

The right column (72, 75, 68, 70, 73) contains the values - the actual data.

At the bottom, dtype: int64 tells us all values are 64-bit integers, just like NumPy arrays.

Why this matters:

Unlike a regular Python list or NumPy array, a Series keeps track of what each value represents. You can access values by their label, not just their position.

Creating a Series with custom labels

# Create a Series with custom index labels
temperatures = pd.Series(
    [72, 75, 68, 70, 73],
    index=['Monday', 'Tuesday', 'Wednesday', 'Thursday', 'Friday']
)
print(temperatures)
Monday       72
Tuesday      75
Wednesday    68
Thursday     70
Friday       73
dtype: int64

Now the index has meaningful labels instead of just positions. This makes the data self-documenting - you can see that 72 is Monday’s temperature without having to remember that position 0 means Monday.

Creating a Series from a dictionary

# Create a Series from a dictionary
# Keys become the index, values become the data
temperatures = pd.Series({
    'Monday': 72,
    'Tuesday': 75,
    'Wednesday': 68,
    'Thursday': 70,
    'Friday': 73
})
print(temperatures)
Monday       72
Tuesday      75
Wednesday    68
Thursday     70
Friday       73
dtype: int64

When you create a Series from a dictionary: - Dictionary keys become the index labels - Dictionary values become the Series values - This is often the most natural way to create a Series with labeled data

Accessing values in a Series

# Access by label
print("Wednesday's temperature:", temperatures['Wednesday'])

# Access by position using .iloc
print("Third day's temperature:", temperatures.iloc[2])

# Access multiple values by label
print("\nWeekday temperatures:")
print(temperatures[['Monday', 'Wednesday', 'Friday']])
Wednesday's temperature: 68
Third day's temperature: 68

Weekday temperatures:
Monday       72
Wednesday    68
Friday       73
dtype: int64

Understanding the access methods:

Label-based access: temperatures['Wednesday'] uses the index label. This is like accessing a dictionary by key - you use the meaningful name.

Position-based access: temperatures.iloc[2] uses the numeric position. .iloc (integer location) explicitly indicates position-based indexing. Position 2 is the third element (Wednesday).

Multiple labels: temperatures[['Monday', 'Wednesday', 'Friday']] passes a list of labels and returns a Series containing just those labeled values. Note the double brackets - the outer brackets are for indexing, the inner brackets create the list of labels.

Vectorized operations on Series

# Convert Fahrenheit to Celsius (remember Week 3 vectorization?)
celsius = (temperatures - 32) * 5/9
print("Temperatures in Celsius:")
print(celsius.round(1))

# Boolean indexing (remember Week 3?)
hot_days = temperatures[temperatures > 72]
print("\nDays warmer than 72°F:")
print(hot_days)
Temperatures in Celsius:
Monday       22.2
Tuesday      23.9
Wednesday    20.0
Thursday     21.1
Friday       22.8
dtype: float64

Days warmer than 72°F:
Tuesday    75
Friday     73
dtype: int64

Understanding vectorized operations:

Just like NumPy arrays from Week 3, you can perform operations on entire Series without writing loops:

Arithmetic operations: (temperatures - 32) * 5/9 subtracts 32 from every temperature and multiplies by 5/9 to convert Fahrenheit to Celsius. The operation applies to all values at once.

Boolean indexing: temperatures > 72 creates a boolean Series (True/False for each day), then temperatures[...] selects only the True values. This is exactly like boolean indexing in NumPy from Week 3.

Key difference from NumPy: The index labels (day names) are preserved through all operations. When you filter or transform the data, the labels stick with their values.

Series attributes

# Explore Series attributes
print("Values (as NumPy array):", temperatures.values)
print("Index labels:", temperatures.index)
print("Data type:", temperatures.dtype)
print("Shape:", temperatures.shape)
print("Size:", temperatures.size)
Values (as NumPy array): [72 75 68 70 73]
Index labels: Index(['Monday', 'Tuesday', 'Wednesday', 'Thursday', 'Friday'], dtype='object')
Data type: int64
Shape: (5,)
Size: 5

Understanding Series attributes:

.values: Returns the underlying NumPy array. This is how you “drop down” to NumPy when you need array-specific operations. The connection between pandas and NumPy is explicit here.

.index: Returns an Index object containing the labels. You can see all the row labels at once.

.dtype: The data type of the values, just like NumPy arrays. All values in a Series must have the same type.

.shape: A tuple showing dimensions. For a Series, this is always a single number in parentheses: (n,) where n is the length.

.size: The total number of elements. For a Series, this is the same as the length.

Part 2: Creating and Using DataFrames

A DataFrame is a two-dimensional labeled data structure with columns. Think of it as a table or spreadsheet where: - Each column is a Series - Rows have labels (the index) - Columns have names - Different columns can contain different data types

Creating a DataFrame from a dictionary

# Create a DataFrame from a dictionary of lists
# Each key becomes a column name, each list becomes the column values
grades = pd.DataFrame({
    'student': ['Alice', 'Bob', 'Charlie', 'Diana', 'Eve'],
    'quiz1': [85, 92, 78, 88, 95],
    'quiz2': [88, 85, 82, 90, 92],
    'final': [90, 88, 85, 92, 94]
})

print(grades)
   student  quiz1  quiz2  final
0    Alice     85     88     90
1      Bob     92     85     88
2  Charlie     78     82     85
3    Diana     88     90     92
4      Eve     95     92     94

Understanding the output:

The leftmost column (0, 1, 2, 3, 4) is the row index - labels for rows, just like a Series. By default, pandas uses sequential integers.

The top row (student, quiz1, quiz2, final) contains column names - labels for columns.

Each cell contains a value. This looks like a spreadsheet or database table.

How dictionaries become DataFrames:

When you pass a dictionary to pd.DataFrame(): - Each dictionary key becomes a column name - Each dictionary value (a list) becomes the data for that column - All lists must be the same length - pandas automatically creates a row index (0, 1, 2, …)

Setting a meaningful index

# Set the 'student' column as the index
grades = grades.set_index('student')
print(grades)
         quiz1  quiz2  final
student                     
Alice       85     88     90
Bob         92     85     88
Charlie     78     82     85
Diana       88     90     92
Eve         95     92     94

What changed:

Before: Student names were in a column, rows were labeled with numbers (0, 1, 2, …)

After: Student names became the row labels (index), and the ‘student’ column disappeared from the data.

Why this matters:

Setting a meaningful index makes data selection more intuitive. Instead of remembering “Alice is row 0,” you can directly access “the row labeled ‘Alice’.”

Understanding list-of-dictionaries format:

This format is common when processing data row-by-row (like Week 2’s parsing workflow): - Each dictionary represents one complete record (row) - Dictionary keys become column names - pandas automatically aligns everything into a table

Compare this to Week 2, where you built a list of dictionaries manually by parsing CSV rows. If you had that list, you could convert it directly to a DataFrame with pd.DataFrame(list_of_dicts).

DataFrames support multiple ways to select data. Understanding these selection methods is fundamental to working with pandas.

Understanding column selection:

Single column with single brackets: grades['quiz1'] returns a Series. This is one column extracted from the DataFrame. The Series has the same index as the DataFrame (student names).

Multiple columns with double brackets: grades[['quiz1', 'quiz2']] returns a DataFrame. The outer brackets are for indexing, the inner brackets create a list of column names. The result is a subset of the original DataFrame with just those columns.

Why the type matters:

When you select a single column, you get a Series - you can use all Series methods and operations. When you select multiple columns, you get a DataFrame - you can use DataFrame methods. Knowing which you’re working with helps you understand what operations are available.

Understanding .loc:

.loc selects data by labels (index names):

Single row: grades.loc['Alice'] returns a Series containing Alice’s data. The Series index becomes the column names (quiz1, quiz2, final).

Multiple rows: grades.loc[['Alice', 'Charlie', 'Eve']] returns a DataFrame with just those rows. Notice the double brackets again - we’re passing a list of labels.

Rows and columns: grades.loc['Alice', ['quiz1', 'quiz2']] selects Alice’s row but only the quiz1 and quiz2 columns. First argument is row labels, second argument is column labels.

The key idea: .loc uses the labels you can see in the printed DataFrame. If you can see “Alice” in the index, you can use 'Alice' with .loc.

Understanding .iloc:

.iloc selects data by integer position (like NumPy arrays or lists):

Position 0: grades.iloc[0] returns the first row (Alice), regardless of index label.

Position slice: grades.iloc[0:3] returns rows at positions 0, 1, and 2 (first three rows).

Position for rows and columns: grades.iloc[0, 0:2] selects the first row and the first two columns.

The key difference from .loc:

  • .loc uses the names/labels: grades.loc['Alice']
  • .iloc uses numeric positions: grades.iloc[0]

Both happen to return the same row here (Alice is at position 0), but they’re fundamentally different. If you later sort the DataFrame, .loc['Alice'] still gets Alice, but .iloc[0] gets whoever is now in the first position.

Part 4: Operations and Automatic Alignment

One of pandas’s most powerful features is automatic alignment by index. When you perform operations between Series or DataFrames, pandas automatically aligns data by their labels.

This is what makes pandas different from NumPy.

Arithmetic with Series

# Create two Series with the same index
monday_sales = pd.Series([100, 150, 200], index=['Product A', 'Product B', 'Product C'])
tuesday_sales = pd.Series([120, 140, 210], index=['Product A', 'Product B', 'Product C'])

print("Monday sales:")
print(monday_sales)
print("\nTuesday sales:")
print(tuesday_sales)

# Add the Series (element-wise, like NumPy)
total_sales = monday_sales + tuesday_sales
print("\nTotal sales (Monday + Tuesday):")
print(total_sales)

Understanding the addition:

When the indices match perfectly, addition works like NumPy: - Product A: 100 + 120 = 220 - Product B: 150 + 140 = 290
- Product C: 200 + 210 = 410

Each value is added to the corresponding value with the same label.

Automatic alignment with mismatched indices

# Create two Series with different (but overlapping) indices
monday_sales = pd.Series([100, 150, 200], 
                         index=['Product A', 'Product B', 'Product C'])

# Tuesday has Products B, C, D (Product A is missing, Product D is new)
tuesday_sales = pd.Series([140, 210, 180], 
                          index=['Product B', 'Product C', 'Product D'])

print("Monday sales:")
print(monday_sales)
print("\nTuesday sales:")
print(tuesday_sales)

# Add the Series - pandas aligns by label automatically
total_sales = monday_sales + tuesday_sales
print("\nTotal sales (Monday + Tuesday):")
print(total_sales)

Product B    150
Product C    200
dtype: int64

Product B    140
Product C    210
Product D    180
dtype: int64

Total sales (Monday + Tuesday):
Product A      NaN
Product B    290.0
Product C    410.0
Product D      NaN
dtype: float64

Understanding automatic alignment:

This is pandas’s superpower. Watch what happens:

Product A: Exists Monday but not Tuesday → Result is NaN (Not a Number, representing missing data)

Product B: Exists both days (150 + 140) → Result is 290

Product C: Exists both days (200 + 210) → Result is 410

Product D: Exists Tuesday but not Monday → Result is NaN

pandas didn’t crash or give you wrong answers. It: 1. Found which labels exist in both Series (B and C) 2. Added those matching values 3. Marked everything else as missing (NaN)

Why this matters:

In real data analysis, datasets rarely align perfectly: - Different time periods might have different products sold - Different stores might track different metrics - Data sources might use different identifiers

NumPy would crash or give you wrong results if array lengths don’t match. pandas automatically handles misalignment by tracking labels.

Compare to NumPy:

This would fail with NumPy because array lengths don’t match

np.array([100, 150, 200]) + np.array([140, 210, 180, 220]) # ERROR!

pandas handles it gracefully by aligning labels

Understanding column operations:

Creating new columns: grades['total_quiz'] = grades['quiz1'] + grades['quiz2'] creates a new column by adding two existing columns. The addition happens element-wise (row by row).

Operations on Series: When you write grades['quiz1'] + grades['quiz2'], you’re adding two Series. Both Series have the same index (student names), so pandas aligns them automatically and adds corresponding values.

Combining operations: (grades['quiz_avg'] * 0.4 + grades['final'] * 0.6) multiplies each column by a scalar, then adds the results. All operations preserve the index alignment.

The result: Each new column has the same index as the original DataFrame, and all calculations align by student name automatically.

Compare to Week 4, where you manually tracked positions and had to be careful that your calculations lined up correctly. pandas handles alignment automatically through the index.

Part 5: Descriptive Statistics and Sorting

pandas provides built-in methods for computing statistics on Series and DataFrames, similar to NumPy but with label preservation.

Computing statistics

# Statistics on a single column (returns a number)
print("Quiz 1 average:", grades['quiz1'].mean())
print("Quiz 1 std dev:", grades['quiz1'].std())
print("Quiz 1 min:", grades['quiz1'].min())
print("Quiz 1 max:", grades['quiz1'].max())

# Statistics on entire DataFrame (returns a Series with one value per column)
print("\nStatistics for all columns:")
print(grades.mean())

Statistics for all columns:

quiz2              87.40

quiz_avg           87.50
weighted_score     88.88

Understanding statistics methods:

On a Series: Methods like .mean(), .std(), .min(), .max() return a single number - the statistic computed across all values.

On a DataFrame: The same methods return a Series with one value per column. grades.mean() computes the mean of each column separately and returns those means as a Series.

Comparison to NumPy: - NumPy: np.mean(array) or array.mean() - pandas: series.mean() or df.mean() - Same concept, but pandas preserves labels

Understanding the axis parameter:

This is the same concept as NumPy’s axis parameter from Week 3:

axis=0: “Collapse the rows” → You get one result per column - Compute down each column separately - Result has the same columns, but only one row

Understanding .sort_values():

Sort by one column: .sort_values('final') sorts the entire DataFrame based on the ‘final’ column. All rows stay together - Charlie’s row moves as a complete unit.

Sort direction: By default, sorting is ascending (lowest to highest). Use ascending=False for descending (highest to lowest).

Sort by multiple columns: .sort_values(['quiz1', 'quiz2']) sorts by quiz1 first, then uses quiz2 to break ties. If two students have the same quiz1 score, they’re ordered by quiz2.

Key point: The entire row moves together. When sorting by final exam score, each student’s quiz scores stay with that student.

What You’ve Learned

This demo introduced pandas’s core concepts:

Series: - One-dimensional labeled arrays - Index provides meaningful labels for values - Supports vectorized operations like NumPy - Can be created from lists, dicts, or NumPy arrays

DataFrames: - Two-dimensional labeled data structures - Each column is a Series - Rows and columns both have labels (indices) - Can be created from dicts, lists of dicts, or other sources

Selection: - Use .loc[label] for label-based selection - Use .iloc[position] for position-based selection - Use boolean indexing to filter rows based on conditions

Automatic Alignment: - pandas aligns data by labels during operations - Mismatched indices result in NaN where labels don’t match - This prevents accidental misalignment errors

Statistics: - Built-in methods like .mean(), .std(), .describe() - Can operate on entire DataFrames or individual columns - Understand axis=0 (column-wise) vs axis=1 (row-wise)

The NumPy Connection

pandas is built on top of NumPy: - Series and DataFrames store data in NumPy arrays internally - Vectorized operations work the same way - You can access the underlying array with .values - The axis parameter works the same way

But pandas adds: - Labels (indices) for rows and columns - Automatic alignment by labels - Mixed data types in different columns - More convenient methods for data manipulation
